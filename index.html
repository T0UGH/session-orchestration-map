Agent Loop A：Plan 回环

Claude plan → Codex review → 不接受点 → 人机讨论 → 修正 plan → reviewer 判断

每个 agent 先形成一套 plan 回环，reviewer 决定是否过关。

Agent Loop B：Plan 回环

同样的回环结构

不同 agent 各自独立跑一套 plan / review / discussion loop。

Agent Loop C：Plan 回环

reviewer 是质量闸门

不是 planner 自己决定是否继续，而是由 reviewer 判断。

Implementation Loop A

新 Codex session 写代码 → 多 reviewer 并行审查 → coder 汇总 → 反驳/吸收 → 修正代码

plan 过关后，agent 直接进入实现，并形成第二组回环。

Implementation Loop B

同样是 coding / review / rebut / modify

单个 agent 内部的实现回环独立收敛。

Implementation Loop C

局部收敛

但局部都收敛，不代表整体收敛。

Global Reviewer

整体看三个 agent

从系统整体视角看三个 agent 的结果，再回环几轮，主要检测跨 agent 的整体问题。

Verification / Conditional Return

联调 / 测试 / 实跑验证按问题性质回流

验证阶段发现问题时，可能回到实现 agent、global reviewer、planner，必要时甚至回到 design 层。
